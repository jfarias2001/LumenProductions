# PRD-011 — Pipeline enxuto + formato no dashboard + visualização de calendário

**Data:** 2026-07-10
**Status:** Aprovado
**Continuação de:** PRD-010

## 1. Motivação (relato do usuário)

Após uso real, o dono do produto pediu três ajustes:

1. **Mostrar o formato da publicação no dashboard** — hoje o card no board não deixa claro se é Reel, imagem única, carrossel ou anúncio; é preciso abrir o card para saber.
2. **Criar uma visualização de calendário** — o calendário editorial já gera peças datadas, mas só as mostra agrupadas por "semana" em lista. Falta enxergá-las numa **grade mensal** de verdade.
3. **Enxugar o pipeline** — várias etapas nunca eram usadas ou eram redundantes. Excluir/mesclar:
   - **Sinais do mercado** (excluir — o fluxo começa em Ideias Brutas)
   - **Direção criativa** (juntar com Roteiro)
   - **Copy/Legenda/CTA** (juntar com Roteiro)
   - **Gravado** (excluir)
   - **Agendado** (excluir)
   - **Em distribuição** (excluir)
   - **Análise** (excluir)
   - **Escalar/Reciclar** (excluir)
   - **Arquivado** (excluir como coluna do board)

## 2. Objetivo

Reduzir o pipeline de **18 → 9 etapas**, tornando Roteiro a etapa que reúne roteiro, direção criativa e copy; exibir o formato da publicação diretamente no board; e oferecer uma visão de calendário mensal para os itens datados do calendário editorial.

## 3. Novo pipeline (9 etapas)

1. Ideias Brutas
2. Ideias Validadas
3. Ângulo Definido
4. Hooks em Teste
5. **Roteiro** (agora reúne roteiro + direção criativa + copy/legenda/CTA)
6. Pronto para Gravar
7. Em Edição
8. Revisão de Retenção
9. Publicado

## 4. Decisões (travadas com o usuário)

- **Etapas excluídas continuam existindo como valores de enum no banco** (não há remoção destrutiva do tipo Postgres). Elas apenas saem do pipeline ativo (board, stepper, gates). Isso evita migração destrutiva e mantém o **arquivar** funcionando (o arquivamento continua marcando `stage = ARQUIVADO` + `archivedAt`, e o board já esconde arquivados por `archivedAt`).
- **Roteiro absorve direção + copy:** a etapa Roteiro passa a exibir as três seções (roteiro, direção criativa, copy) e a "geração com IA" da fase produz as três de uma vez. O gate para avançar exige roteiro completo + formato de direção + legenda/CTA.
- **Gravado excluído:** de Pronto para Gravar segue-se direto para Em Edição (gate = checklist de pré-produção). O link da captação bruta vira campo opcional na etapa Em Edição.
- **Agendado excluído:** de Revisão de Retenção segue-se direto para Publicado (gate = retenção aprovada). Publicar continua manual e sob permissão.
- **Dashboard:** cada card do board passa a exibir um selo de formato: `Reel`, `Imagem única`, `Carrossel` e, quando anúncio, `📣 Anúncio`.
- **Visualização de calendário:** grade mensal dentro do detalhe do calendário editorial (alternância Lista ↔ Calendário), com navegação de mês e cada peça no dia correspondente mostrando pilar/formato/anúncio.

## 5. Fora de escopo

- Remover de fato os valores de enum do banco (mantidos por segurança/compat).
- Reescrever analytics (as métricas de etapas removidas simplesmente ficam vazias).
- Calendário global de múltiplos calendários / cards do pipeline por data.

## 6. Critérios de aceite

- O board mostra 9 colunas (sem Sinais/Direção/Copy/Gravado/Agendado/Distribuição/Análise/Reciclar/Arquivado) e cada card exibe o formato da publicação.
- Um card avança Ideias Brutas → … → Publicado respeitando os novos gates; a auto-produção do calendário leva o card até Pronto para Gravar.
- No calendário editorial é possível alternar para uma grade mensal e ver cada peça no seu dia.
- Arquivar um card continua funcionando.
