# PRD-002 — Redesign do Board + Ativação da Camada de IA

> Continuação do PRD-001. Foco: tornar o produto **usável de ponta a ponta** (criar e operar cards pela interface) e **ativar a camada de IA copiloto** que estava apenas estruturada (Fase 2).

## 1. Contexto / problema

O sistema entregue na Fase 0/1 está tecnicamente correto, mas na prática o usuário percebe duas lacunas graves:

1. **"Só tem um Kanban que não consigo interagir."** O board não oferece forma de **criar cards** pela UI (só os vindos do seed), os filtros não estão expostos e metade das abas do card está "em construção". O resultado é uma tela que parece estática.
2. **A IA não funciona.** Todas as rotas `/ai/*` retornam `503` (stubs). A promessa central do produto — *copiloto de IA acoplado a cada etapa* (PRD-001 §10) — não está acessível.

Além disso, o visual é um cinza/branco plano que não comunica um produto de software profissional.

## 2. Objetivos

- **O1.** Operar o pipeline inteiro pela interface: criar, filtrar, mover, editar e arquivar cards sem tocar no banco.
- **O2.** Ativar as **6 funções de IA** do PRD-001 §10 com OpenAI real, dentro do fluxo de cada etapa, com revisão humana obrigatória nos gates.
- **O3.** Reformular a identidade visual para um **tema dark SaaS moderno** (alto contraste, acentos da marca Lumen, densidade adequada a um board operacional).

## 3. Não-objetivos

- Publicação automática (continua manual — PRD-001).
- Coleta automática de métricas (continua manual).
- Jobs assíncronos via BullMQ para IA: nesta entrega as chamadas de IA são **síncronas** com registro de `AIJob` para observabilidade; fila assíncrona fica como evolução.

## 4. Usuários e valor

| Papel | Ganho |
|---|---|
| Estrategista | Prospecta sinais, estrutura inputs soltos e gera ângulos/hooks com IA. |
| Roteirista | Gera roteiro + copy (Regra de Ouro) como rascunho editável. |
| Gestor/Revisor | Validação assistida por IA como **sugestão** que um humano confirma no gate. |
| Todos | Criar e operar cards visualmente; board legível e agradável. |

## 5. Escopo funcional

### 5.1 Operação do board (O1)
- Botão **+ Novo Card** → modal (título, estágio inicial, pilar, fonte/conteúdo de sinal, responsável).
- Barra de **filtros** (busca por título, pilar, nível de consciência, classe).
- Mover por drag-and-drop respeitando gates (já existe); **arquivar** pelo detalhe.
- Abas do card antes "em construção" passam a funcionar (Copy, Direção, Checklists, Retenção, Agendamento, Reciclagem) ao menos no nível de leitura/edição essencial.

### 5.2 IA copiloto (O2) — mapeamento PRD-001 §10
| Função | Onde aparece | Saída |
|---|---|---|
| Prospecção | Coluna Sinais (seleção de sinais) | ideias → novos cards em IDEIAS_BRUTAS |
| Estruturação | Aba Template | preenche campos + pilar sugerido |
| Validação assistida | Aba Validação | 6 notas (0–3) + justificativas como **sugestão** (`aiSuggested`) |
| Ângulos & hooks | Aba Ângulos | grava ângulos + 5–10 hooks como rascunho |
| Copy | Aba Copy/Roteiro | roteiro {dor,quebra,mecanismo,benefício,cta} + legenda + CTAs |
| Reciclagem | Aba Reciclagem | derivados (carrossel, e-mail, SDR, LinkedIn, hooks novos…) |

**Regras de IA (PRD-001 §10):** chave só no backend; toda saída é rascunho/sugestão editável; nada avança gate sem humano; em falha, fallback "IA indisponível — preencha manualmente".

### 5.3 Identidade visual (O3)
- Tema dark, acentos índigo/marca Lumen, tipografia Inter, cards densos com hierarquia clara, badges de pilar/classe/veredito coloridos, microinterações de hover/drag, estados de loading da IA.

## 6. Critérios de aceite

- [ ] É possível criar um card e vê-lo aparecer no board em tempo real.
- [ ] Filtros alteram o board.
- [ ] Cada uma das 6 funções de IA retorna conteúdo real da OpenAI e persiste no card.
- [ ] Sem `OPENAI_API_KEY`, a UI mostra fallback claro em vez de quebrar.
- [ ] Validação por IA entra como sugestão e exige confirmação humana antes do gate (regra de `PipelineService` preservada).
- [ ] Visual dark aplicado de forma consistente em login, board e detalhe.

## 7. Riscos

- Custo/latência de IA → modelo configurável (`AI_DEFAULT_MODEL`, default `gpt-4o-mini`), temperatura baixa, saída estruturada.
- Prompt injection → todo texto do usuário tratado como dado; saída validada por Zod (descarta fora do schema).
